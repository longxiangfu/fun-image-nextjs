# 🔧 FaceMesh问题快速修复指南

## 🚨 问题：faceLandmarks为null

### 症状
点击眼睛时控制台显示：
```
faceLandmarks: null
没有面部关键点数据
未知区域，触发创意动画
```

### 🎯 解决方案

### 方案1：刷新页面重试（推荐）
1. **按F5刷新页面**
2. **等待5-10秒**让MediaPipe完全加载
3. **重新上传图片**
4. **点击眼睛测试**

### 方案2：检查网络连接
1. **打开浏览器开发者工具（F12）**
2. **切换到Network标签**
3. **查找FaceMesh脚本加载状态**
4. **确保没有网络错误**

### 方案3：使用更好的测试图片
**推荐图片特征**:
- ✅ 清晰的正脸照片
- ✅ 光线充足
- ✅ 人脸完整可见
- ✅ 分辨率适中（800x600以上）

**避免**:
- ❌ 模糊的照片
- ❌ 侧脸或半脸
- ❌ 戴眼镜
- ❌ 光线太暗

### 方案4：查看详细调试信息
**打开控制台（F12）查看**:
```
开始初始化FaceMesh...
FaceMesh脚本已加载
创建FaceMesh实例...
FaceMesh初始化完成
```

**应该看到**: 所有步骤都成功完成

**如果看到错误**: 查看具体错误信息

## 🔍 故障排除

### 问题：控制台显示"MediaPipe FaceMesh not loaded"
**原因**: CDN脚本加载失败
**解决**:
- 检查网络连接
- 刷新页面重试
- 检查防火墙设置

### 问题：控制台显示"FaceMesh init failed"
**原因**: 实例创建失败
**解决**:
- 查看具体错误信息
- 刷新页面重试
- 检查浏览器兼容性

### 问题：控制台显示"未检测到面部关键点"
**原因**: MediaPipe未能检测到人脸
**解决**:
- 使用更清晰的人脸照片
- 确保光线充足
- 尝试不同角度的照片

### 问题：控制台显示"FaceMesh未就绪"
**原因**: 初始化未完成
**解决**:
- 已添加1秒延迟自动处理
- 等待几秒后重试
- 刷新页面重新初始化

## 🚨 如果仍然不工作

### 最后的手段：完全重置

1. **清除浏览器缓存**
   - 按 Ctrl+Shift+Delete
   - 清除缓存和Cookie
   - 重新加载页面

2. **使用隐私/无痕模式**
   - 打开隐私模式
   - 访问 http://localhost:3000
   - 测试是否正常

3. **重启开发服务器**
   ```bash
   # 停止当前服务器
   # 重新启动
   npm run dev
   ```

4. **检查控制台错误**
   - 查看是否有JavaScript错误
   - 查看网络请求是否失败
   - 查看MediaPipe脚本是否加载
