## 🔍 如果不工作，检查以下几点

### 检查1：没有faceLandmarks数据
**症状**：控制台显示 `faceLandmarks: false`
**原因**：MediaPipe没有检测到人脸
**解决**：
- 确保图片清晰可见人脸
- 尝试其他包含人脸的图片
- 检查网络连接（MediaPipe需要从CDN加载）

### 检查2：所有区域点击检查都是false
**症状**：控制台显示所有区域的 `点击检查: false`
**原因**：点击位置不在识别区域内
**解决**：
- 尝试点击眼睛的中心位置
- 确保点击在图片范围内
- 眼睛区域现在有25px的padding，应该更容易点击

### 检查3：显示"未知区域，触发创意动画"
**症状**：控制台显示 `❌ 没有匹配到任何面部区域` 然后触发创意动画
**原因**：点击位置没有匹配到任何面部区域
**解决**：
- 确认AI确实检测到了人脸
- 尝试点击更明显的面部特征
- 查看控制台的边界信息

### 检查4：显示"执行眼睛眨眼动画"但没有视觉效果
**症状**：控制台显示动画触发，但看不到效果
**原因**：CSS动画或DOM元素问题
**解决**：
- 检查 Elements 标签中是否有 `region-anim` 元素
- 确认 CSS 动画定义正确
- 检查 z-index 是否正确

## 📸 推荐测试图片

使用以下类型的图片测试：

1. **正面人脸照片** - 最容易识别
2. **清晰的自拍** - 眼睛区域明显
3. **证件照** - 人脸标准角度
4. **艺术照** - 测试不同风格

避免使用：
- 模糊的照片
- 侧脸照片（眼睛可能被遮挡）
- 戴眼镜的照片（可能影响识别）
- 光线太暗的照片

## 💡 测试技巧

1. **点击中心**：尽量点击眼睛的正中心，而不是边缘
2. **等待识别**：确保AI识别完成后再点击
3. **多次尝试**：在不同位置点击，找到最佳识别点
4. **查看日志**：控制台的日志会告诉你具体的识别情况

## 🚨 常见错误信息

### 错误1：`没有面部关键点数据`
**含义**：MediaPipe没有检测到人脸
**解决**：上传更清晰的人脸照片

### 错误2：`❌ 没有匹配到任何面部区域`
**含义**：点击位置不在任何识别区域内
**解决**：尝试点击眼睛的中心位置

### 错误3：`未找到imageWrapper元素`
**含义**：DOM元素不存在
**解决**：刷新页面重新测试

### 错误4：`未知的面部区域: xxx`
**含义**：region名称不匹配
**解决**：检查代码中的region名称是否一致
